=======================INFORAMCION DEL PROYECTO ============================

ESTE PROYECTO HA SIDO CONSTRUIDO BAJO EL PATRON DE DISEÑO MVC (MODELO - VISTA - CONTROLADOR), 
ES UN PATRON DE DISEÑO EN EL CUAL LA VISTA SOLICITA AL CONTROLADOR UNA INFORMACION, 
EL CONTROLADOR SE COMUNICA CON EL MODELO (EL CUAL ES UN ESPEJO DE LAS TABLAS DE LA BASE DE DATOS) Y A SU 
VEZ EL MODELO LE RESPONDE AL CONTROLADOR CON LA INFORMACION REQUERIDA DE LA 
BASE DE DATOS, ESTO ES UNA COMUNICACION CONSTANTE ENTRE ESTOS 3 ELEMENTOS. 

ADICIONAL A ESO SE USO EL PARADIGMA DE PROGRAMACION LLAMADO 'ORIENTADO A OBJETOS', EN ESTE SE USAN CLASES 
PARA CONTENER METODOS Y ATRIBUTOS QUE PUEDEN SER HEREDADAS A OTRAS CLASES,
POR EJEMPLO, EN LA CARPETA MODELS, HAY UN ARCHIVO LLAMADO 'CLS_DB.PHP', ESTA ES LA CLASE PADRE PARA LA 
CONEXION A LA BASE DE DATOS, CONTIENE ATRIBUTOS Y METODOS PRIVADOS  O PROTEJIDOS,
LOS CUALES SOLO SE PUEDEN USAR DENTRO DE LA CLASE 'CLS_DB' (EN EL CASO DE ATRIBUTOS O METODOS PRIVADOS), O ESTOS PUEDEN SER HEREDADOS A LAS DEMAS CLASES DE LA CARPETA MODELS 
(EN EL CASO DE ATRIBUTOS Y METODOS PROTEJIDOS), DE ESTA MANERA SE HACE LA CONEXION A LA BASE DE DATOS UNA VEZ, Y EL RESTO DE CLASES PUEDEN HACER USO DE ESTA

======================= ARCHIVOS ==========================================

LAS CARPETAS SON (CONTROLLERS, MODELS, VIEWS)
LOS ARCHIVOS PRINCIPALES SON (INDEX.PHP, .HTACCESS)

.HTACCESS = ES UN ARCHIVO DE CONFIGURACION DE RUTAS PARA EL SERVIDOR XAMPP - WAMPP, EL QUE SEA PUES, ES UN ARCHIVO MUY IMPORTANTE Y NO PUEDE FALTAR

INDEX.PHP = ES DE IGUAL MODO, OTRO ARCHIVO AUN MAS IMPORTANTE, TODO LO QUE PASA EN EL PROYECTO, EJECUTA DENTRO DE INDEX.PHP
